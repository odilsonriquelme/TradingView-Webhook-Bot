DESCRIÇÃO
---------
Servidor web (Flask) que escuta ALERTAS DO TRADINGVIEW via webhook e os redistribui
instantaneamente para múltiplos canais de notificação. Tudo configurável por canal.

FLUXO:
  TradingView dispara alerta -> POST em http://<IP>/webhook -> main.py valida a "key"
  (precisa bater com sec_key no config.py) -> handler.send_alert() distribui aos canais ativos.

CANAIS SUPORTADOS (confirmado no handler.py)
---------------------------------------------
- Telegram  (lib python-telegram-bot — telegram.Bot)
- Discord   (discord_webhook — DiscordWebhook/DiscordEmbed)
- Slack     (slack_webhook — Slack)
- Twitter/X (tweepy — OAuthHandler/API)
- Email     (smtplib + email.mime.text)
Cada canal é ligado/desligado individualmente no config.py (send_*_alerts).
Suporta variáveis do TradingView no texto ({{ticker}}, {{close}}, {{exchange}}, etc.)
e roteamento dinâmico (o próprio alerta pode indicar telegram/discord/slack de destino).

ARQUIVOS E SUAS FUNÇÕES
------------------------

                 Twitter/Email). (97 linhas)

                 slack-webhook, waitress).
Dockerfile / docker-compose.yml → containerização (docker-compose build && up).
assets/        → logo.png e bmac.png (imagens do README do fabston).
